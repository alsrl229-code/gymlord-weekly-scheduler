# 짐로드 주간 스케줄 관리도구

`index.html`을 브라우저로 열면 바로 사용할 수 있습니다. GitHub Pages와 Supabase를 연결하면 모바일과 데스크톱에서 같은 데이터를 로그인 기반으로 동기화할 수 있습니다.

## 기본 흐름

1. `회원표`에서 회원명, 유형, 고정 일정, 가능 범위를 입력합니다.
2. 회원별 `확보분`을 입력합니다. 기본값은 80분이며 시간 겹침 계산에 사용됩니다.
3. 부부처럼 같은 슬롯을 공유하는 회원은 `공유그룹`에 같은 이름을 입력합니다.
4. `카톡작업`에서 회원별 답장 문구를 만들고, 제안/최종 일정의 겹침 여부를 확인합니다.
5. `카톡문구`에서 회원별 확인 문구를 복사해 카톡으로 보냅니다.
6. `응답정리`에서 확정, 변경요청, 변경반영, 취소 상태를 정리합니다.
7. `브로제이`에서 등록할 최종 목록을 복사하거나 CSV로 저장합니다.

## 일정 입력 형식

- `월 19:00, 목 20:00`
- `화 오후 8시`
- `7/2 20:00`
- `7월 2일 오후 8시`

## 시간 겹침 체크

회원별 `확보분` 기준으로 시작-종료 시간을 계산합니다. 예를 들어 `13:00` 시작에 확보분이 `80`이면 `13:00-14:20`으로 계산되어, `14:00` 시작 일정과 겹침으로 표시됩니다.

## 이번주 예외 확보분

`카톡작업` 탭의 제안/최종 일정 아래에서 특정 일정만 이번 주 확보분을 바꿀 수 있습니다. 예를 들어 기본 80분 회원도 `금 07:00`만 `60분`으로 바꾸면 해당 일정은 `07:00-08:00`으로 겹침 계산됩니다.

현장에서 직접 확인해서 괜찮은 겹침은 `이번주 겹침 경고 숨김`을 체크하면 해당 슬롯의 경고만 숨길 수 있습니다. 기본 회원 확보분은 바뀌지 않습니다.

## 공유그룹

부부, 가족처럼 같은 예약 슬롯을 공유하는 회원은 `회원표`의 `공유그룹` 칸에 같은 값을 입력합니다. 같은 공유그룹 안의 회원끼리는 서로 시간 겹침으로 표시하지 않습니다.

예: `예시회원 공유A`, `예시회원 공유B`는 `예시공유그룹`으로 묶여 있습니다. 같은 공유그룹 안의 회원끼리는 서로 시간 겹침으로 표시하지 않습니다.

## 회원 답변 자동 등록

`카톡작업` 탭에서 회원이 보낸 답변을 붙여넣고 `답변에서 일정 만들기`를 누르면 `화 13:00, 목 14:00` 같은 최종 일정 문장으로 변환됩니다. 이후 `최종일정 반영`을 누르면 응답정리와 브로제이 등록 리스트에 반영됩니다.

## 카톡 문구 저장

`카톡작업` 또는 `카톡문구` 탭에서 문구를 직접 수정한 뒤 `문구 저장`을 누르면 해당 주차의 회원별 저장문구로 유지됩니다. `기본문구`를 누르면 자동 생성 문구로 되돌아갑니다.

## 주차 이력

주차별 응답정리 내용은 자동으로 따로 저장됩니다. `이력` 탭에서 이전 주차를 선택하면 회원별 상태, 최종 일정, 요청 내용, 등록 메모, 저장문구 여부, 이번주 예외 확보분 기록을 확인할 수 있습니다.

필요하면 `이력 복사`로 표를 복사하거나 `이력 CSV`로 저장할 수 있습니다.

## 로그인/클라우드 저장

기본 상태에서는 브라우저 로컬 저장소에 저장됩니다. `config.js`에 Supabase URL과 anon/public key를 넣으면 상단에 Google/이메일 로그인 패널이 활성화되고, 로그인한 계정의 클라우드 데이터와 동기화됩니다.

공개 GitHub 저장소에는 실제 회원 정보가 들어가지 않도록 기본 회원은 예시 데이터로 되어 있습니다. 실회원 초기 데이터는 로컬 전용 `private/짐로드_실회원_초기데이터.private.json`에 백업되어 있으며, 배포 후 로그인한 상태에서 `불러오기`로 가져와 클라우드에 저장하면 됩니다.

배포 순서는 `DEPLOY.md`를 확인하세요.

## 데이터 저장

브라우저의 로컬 저장소에 자동 저장됩니다. 컴퓨터를 바꾸거나 백업이 필요하면 상단의 `내보내기`로 JSON 파일을 저장하고, 다음에 `불러오기`로 복원하면 됩니다.
